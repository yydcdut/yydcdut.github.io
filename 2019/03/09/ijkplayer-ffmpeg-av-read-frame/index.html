ijkplayer框架简析 -- av_read_frame

2019年3月9日

av_read_frame() 的作用是读取码流中的音频若干帧或者视频一帧

av_read_frame

av_read_frame() 会先判断未解码缓存中是否又数据，如果有的话则调用 read_from_packet_buffer() ，否则就调用 read_frame_internal()

read_frame_internal

  • 调用了 ff_read_packet() 从相应的 AVInputFormat 读取数据
  • 如果媒体频流需要使用 AVCodecParser，则调用 parse_packet() 解析相应的 AVPacket

ff_read_packet

在 for 循环中调用了 iformat->read_packet(s, pkt) ，是 AVInputFormat 的指针函数，以 mp4 为例对应的是

  • mov_find_next_sample() 找到要读取的文件的偏移 + size
  • avio_seek(sc->pb, sample->pos, SEEK_SET) 将文件的指针定位到文件的偏移处
  • av_get_packet(sc->pb, pkt, sample->size) 从文件的偏移读取 size 的数据

parse_packet

parse_packet() 的作用是给需要 AVCodecParser 的媒体流提供解析 AVPacket 的功能，主要是通过 av_parser_parse2() 来解析 AVPacket
